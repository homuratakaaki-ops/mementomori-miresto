## 1. 概要

メメントモリのプラチナガチャ／ピックアップガチャの抽選結果を、所持チケット・所持ダイヤ・セレクト設定・ピックアップ天井カウントをもとに再現する単一ページツール。

2026-07-15 の commit `07b4189` で、資源不足時の抽選打ち切り、localStorage 保存タイミング、HTML エスケープの仕様を確定した。

## 2. 資源消費・打ち切り仕様（2026-07-15確定）

- シミュレーターは「手持ちで回せる範囲を再現」する。
- 要求回数に対し、資源（チケット→ダイヤの優先順）が尽きた時点で抽選を打ち切る。
- 資源チェックは各回の抽選前に行う。
- 打ち切られた回では天井カウントは進まない。
- ダイヤはループ内で実残高から減算する。後計上ではない。
- 不足ダイヤの定義は `未実行回数 × cost_per_pull` とする。
- 不足ダイヤは、要求回数を完走するための追加必要数を表す。
- 対象ガチャのチケットが0、かつダイヤが1回分未満の場合は、抽選せず実行前エラーを表示する。
- 打ち切り発生時は `資源不足のため N回で停止しました（残りM回にはダイヤX個必要）` を表示する。
- 総抽選回数は、要求回数ではなく実行分のみ計上する。

## 3. 状態管理・永続化仕様

- localStorage には実行後の状態を保存する。
- 保存対象は、チケット残、ダイヤ残、天井カウント、セレクト状態、要求回数などの継続用状態とする。
- リロード時は、最後に保存された実行後の値が復元される。
- `lastInputSnapshot` は実行前スナップショットとして保持する。
- `lastInputSnapshot` は「結果をリセット」の戻り先であり、localStorage とは役割を分離する。
- 「結果をリセット」実行時は、復元した実行前状態を localStorage にも保存する。
- これにより、リセット直後にリロードしても画面表示と永続化状態が一致する。
- タブ切替時は、切替元タブの現在状態を localStorage に保存してから切り替える。

## 4. セキュリティ

- ユーザー入力（ピックアップキャラ名）およびキャラ名などの動的文字列は、innerHTML 挿入前に必ず `escapeHtml` を通す。
- `escapeHtml` は `&`, `<`, `>`, `"`, `'` の5文字を変換対象とする。
- 今後の機能追加時も、ユーザー入力またはデータ由来の動的文字列を HTML として直接挿入しない。

## 5. 受け入れ確認

commit `07b4189` の反映時に、以下を確認済み。

- チケット3・ダイヤ0で10連を要求した場合、3回で停止する。
- 上記ケースの不足ダイヤは `7 × cost_per_pull` となる。
- プラチナガチャでは `cost_per_pull = 300` のため、不足ダイヤは2,100個となる。
- 実行後リロードでは、チケット・ダイヤが実行後の値で復元される。
- 「結果をリセット」後のリロードでは、チケット・ダイヤ・天井カウントが実行前の値で復元される。
- ピックアップ名に `<b>test</b>` を入力しても、SR履歴ではタグではなく文字列として表示される。
