# 進化必要枚数計算ツール 仕様書(フェーズ1)

最終更新: 2026-07-14
ステータス: 実装済み・QA修正反映済み(検収合格: 4ef1f1a、2026-07-14)
対象サイト: memento.musoudc.com
ファイル名(案): `evolution-calc.html`(ルート直下、dedicated-weapon-calc.html と同列)

---

## 1. 概要

現在レアリティから目標レアリティまでに必要な同名キャラ・同属性素材の枚数を計算し、
所持数を差し引いた不足数を表示する単一ページツール。

- 想定ユーザー: 無課金・微課金の初心者〜中級者
- 初期SRキャラを素材に回せるほどキャラが余っている層は対象外
- ダイヤ換算・到達日数の試算は本ツールに持たせない(ガチャツール側の守備範囲)

## 2. スコープ

### フェーズ1(本仕様の範囲)

- 起点は SR 固定(N・R起点は対象外)
- 目標レアリティ: SR+ / SSR / SSR+ / UR / UR+ / LR / LR+1〜LR+5
- 必要枚数を「同名キャラ」と「同属性素材」に分けて表示
- 同属性素材は R キャラ何枚から育成する必要があるかまで分解して表示
- 所持数入力(同名・同属性素材・R素材)から不足数を算出
- ガチャツールへの受け渡し口(URLパラメータ)の実装(§6)
- localStorage 保存は不要(入力が少なく毎回入力で成立するため)

### フェーズ2(本仕様の範囲外・実装しない)

- 無料入手ペースからの到達日数試算
- 進化素材の不足状況を考慮したピックアップ/運命ガチャの優先度提案
  (→ ガチャツール側で実装。IDEAS.md カテゴリ5へ追記)
- ガチャツール側での `need` パラメータ受け取り実装

## 3. 入力項目

| 項目 | 形式 | 備考 |
|---|---|---|
| 目標レアリティ | セレクト | SR+〜LR+5 |
| 所持: 同名キャラ | 数値 | 育成対象本体を除いた枚数 |
| 所持: 同属性SR+素材 | 数値 | 必要段階が異なるため SSR+ と分離(as-built) |
| 所持: 同属性SSR+素材 | 数値 | 同上 |
| 所持: R キャラ(同属性) | 数値 | 素材化前の R 枚数 |

### R不足の計算仕様(2026-07-14 確定・4ef1f1a)

- 所持SSR+素材の充当は必要数まで。**余剰SSR+はSR+不足に充当しない**(SSR+はSR+に戻せないため)
- 余剰SR+素材のみ 18枚/体 でR充当可能数に加算(上方向充当は追加R込みの換算として正確)
- `creditR = ownedR + max(0, ownedSrPlus − needSrPlus) × 18`
- `rShortage = max(0, srPlusShortage×18 + ssrPlusShortage×72 − creditR)`

- 現在レアリティは SR 固定のため入力欄を設けない(表記のみ)
- 属性選択は計算に影響しないため不要。「同属性で数える」旨を注記する

## 4. 出力項目

1. 必要な同名キャラ 総数 / 不足数
2. 必要な同属性素材 総数 / 不足数(段階別内訳つき)
3. 同属性素材を作るために必要な R キャラ換算枚数 / 不足数(R充当可能数を加味)
4. 不足がある場合のみ「ガチャ計算ツールで必要ダイヤを見る」ボタン(§6)

不足数が 0 以下の場合は「素材は足りています」を明示する。

## 5. 進化テーブル(共通データ・確定値)

**数値は 2026-07-14 に夢爽が確認済みの確定値。ミコトは変更しないこと。**

- 対象は通常4属性(藍・紅・翠・黄)のみ。**天・冥属性は本ツール対象外**
  (進化素材が同名キャラのみで体系が異なる。ガチャツール側でカバー済み)。
  ツール上部に「天冥属性は対象外」の注記を表示する。

共通データとして `data/evolution-table.json` に切り出し、
本ツールと将来のガチャツール連携の双方から参照する(IDEAS.md カテゴリ15「更新箇所を1か所に集約」に準拠)。

### 換算ルール

- 同名SR+ 1体 = 同名SRキャラ 2枚
- 同属性SR+素材 1体 = 同属性Rキャラ 18枚
- 同属性SSR+素材 1体 = 同属性Rキャラ 72枚
- RキャラはSSR+までしか進化できない(UR不可)

### 検算用の累計値(QAで使用)

- SR→LR: 同名SR 8枚(進化元含む)、同属性R 180枚(SR+×2=36、SSR+×1×2回=144)
- SR→LR+5: 同名SR 20枚(進化元含む)

### 目標レアリティ範囲について

当初フェーズ1は SR+〜LR を対象としていたが、確定値が LR+5 まで揃ったため **SR+〜LR+5** に拡張した(§2・§3は修正済み)。

## 6. ガチャツール連携(受け渡し口)

- 不足する同名キャラ数を URL パラメータで渡す:
  `gacha-calc.html?need={不足枚数}`
- フェーズ1では**リンクを張るのみ**。ガチャツール側の受け取り実装はフェーズ2
- localStorage 経由は採用しない(壊れにくさと単純さを優先)
- 前例: dedicated-weapon-calc.html の「必要魔水晶をガチャ計画へ反映」と同パターン

## 7. UI 方針

- 1画面構成。入力ブロック → 計算結果ブロックの縦並び
- モバイル前提(既存ツールと同様)
- 計算はボタン押下または入力即時のどちらでも可(既存ツールの流儀に合わせる)
- 外部CDN依存は追加しない

## 8. QA 観点(最低限)

- 目標レアリティ全パターンで必要枚数がテーブル通りか
- 所持数 > 必要数 のとき不足 0 表示になるか
- 所持数に負数・空欄・巨大値を入れたときの挙動
- `?need=` の値が不足数と一致するか
